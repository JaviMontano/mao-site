Dos Mundos de Prompting

No es lo mismo hablar con un chatbot que dirigir un agente. Cada modo tiene su logica.

Chatbots: Output-Driven

Usas ChatGPT, Claude o Gemini para producir un entregable concreto. Tu estas al mando de cada paso.

Ideal para: documentos, emails, analisis, contenido

Agentes: Outcome-Driven

Diriges un agente que opera con autonomia. Tu defines el resultado, el agente disena el camino.

Ideal para: sistemas, pipelines, automatizaciones

Meta-Prompting

Si el prompting crea valor una vez, el meta-prompting crea valor infinitas veces. No es escribir prompts — es disenar el sistema que los genera.

9 Principios del Prompting de Alto Rendimiento

1. Rol Hibrido y Acreditado

Todo prompt empieza con un arquetipo experto que combina disciplinas y credenciales reales. El rol calibra tono, profundidad y metodologia.

2. Contexto Prospectivo

La Situacion define el "para que", no el "por que". Que se desbloquea, quienes son los stakeholders, que restricciones existen.

3. Tarea Orientada a Entregable

La mision se define en torno a un entregable concreto: documento, tabla, analisis, plan. Especifico, medible, accionable.

4. Estructura Completa (SPEC)

Situacion + Pedido + Ejecucion + Criterio + [inputs] + [checklist]. Nada queda implicito.

5. Protocolo Metacognitivo

La IA primero interpreta > planifica > ejecuta. Previene outputs superficiales y garantiza co-creacion.

6. Parametrizar para Reutilizar

Usa {{PLACEHOLDERS}}. Un prompt parametrizado se ejecuta 100 veces con diferentes inputs; uno fijo, solo una.

7. Prompt Chaining

Para entregables complejos, encadena prompts: el output de uno es el input del siguiente. Cada prompt hace UNA cosa bien.

8. Pipeline 0-9

El proceso estandar de 10 pasos: Priming → Entender → Definir → Planificar → Ejecutar → Robustecer → Simplificar → Validar → Entregar → Cristalizar.

9. Cristalizar para Escalar

Despues de cada buen resultado, haz ingenieria inversa: extrae un prompt reutilizable. Cada sesion produce el entregable Y el prompt que lo replica.

10. Prompt Stacking

Inserta un prompt dentro de otro. El externo orquesta, el interno ejecuta. Crea sistemas recursivos: prompts que generan prompts.

El Formato SPEC

SPEC no es un template — es un contrato de co-creacion entre tu intencion y la capacidad de la IA.

S Situacion

Para que? Define que se desbloquea, stakeholders, restricciones, y que pasa si NO se hace. Sin Situacion, la IA inventa contexto.

P Pedido

Que necesito? El arquetipo experto (rol + credenciales) y la especificacion del entregable (componentes, estructura, alcance).

E Ejecucion

Como se hace? Protocolo MetodologIA (Interpreta > Planifica > Ejecuta) + frameworks a aplicar + decisiones que requieren validacion humana.

C Criterio

Como se que esta listo? Formato, tono, audiencia, accion esperada + [checklist] de auto-auditoria. El quality gate integrado.

Ejemplo: SPEC Completo

[inputs] SECTOR: {{SECTOR}} COMPETIDORES: {{COMPETIDORES}} --- S | SITUACION --- Para tomar decisiones de posicionamiento en {{SECTOR}}. La inteligencia desbloquea decisiones de inversion. --- P | PEDIDO --- Arquetipo: Analista Senior de Inteligencia Competitiva con MBA y certificacion SCIP (15+ anos). Producir Dossier: resumen ejecutivo + scoring + tabla. --- E | EJECUCION --- Protocolo MetodologIA: Interpreta > Planifica > Ejecuta. Metodo: Porter + PESTEL + scoring comparativo. --- C | CRITERIO --- Max 10 pags. Tono: ejecutivo. Audiencia: director. [checklist] ☐ Scoring numerico por competidor ☐ Min. 5 fuentes verificables ☐ Recomendaciones accionables

4 Clausulas Universales

Cada SPEC hereda 4 clausulas que remueven friccion sin que el usuario las pida.

Perfil

Usa tu contexto (rol, industria, pais) sin repetirlo. Espanol LatAm. Terminos tecnicos en ingles.

Sensorial

Detecta adjuntos automaticamente (texto, imagen, datos, codigo). Extrae, analiza, confirma.

Capacidades

Activa search, code interpreter, vision proactivamente. Usa la herramienta correcta cuando el SPEC lo requiere.

Memoria

Al finalizar, genera KEY FACTS: entidades, decisiones, numeros, proximos pasos. Copy-paste para la siguiente sesion.

Las clausulas se inyectan automaticamente — la friccion desaparece por diseno.

Que es un AI Native Workflow

Un flujo de trabajo disenado CON IA desde el origen — no un proceso manual al que le "agregan IA" despues.

El humano disena, dirige y valida. La IA investiga, genera, itera y ejecuta.

Encadenado: output de un paso = input del siguiente
Multimodal: mezcla chat, voz, video, agentes
Asincrono: algunos pasos toman minutos — paralelizas
Auto-mejora: cada ejecucion cristaliza prompts reutilizables

Tu Primer SPEC

La mejor forma de aprender es hacer. 10 minutos, 3 pasos:

1. Elige un problema real

Algo de esta semana. Un email, un analisis, una presentacion. No un ejercicio teorico.

2. Aplica el formato SPEC

Escribe las 4 secciones: S (para que), P (que + quien), E (como), C (criterio).

3. Ejecuta y evalua

Pega en ChatGPT/Claude/Gemini. El output fue profesional? Listo para usar sin edicion?

Si no, ajusta tu SPEC y repite. Cada iteracion te hace mejor.
