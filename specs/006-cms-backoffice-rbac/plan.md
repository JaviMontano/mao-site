# Implementation Plan: CMS Backoffice RBAC

**Branch**: `006-cms-backoffice-rbac` | **Date**: 2026-03-23 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/006-cms-backoffice-rbac/spec.md`

## Summary

Transform the binary-admin CMS into a 4-role RBAC system with user management, domain allowlisting, audit viewer, and content version restore. Fix the Firebase import error (BUG-003) by bundling with esbuild. Add Cloud Functions for privileged operations (role assignment, auto-provisioning). Extend the admin UI with new role-gated tabs.

## Technical Context

**Language/Version**: JavaScript ES2022 (browser + Node.js 20 for Cloud Functions)
**Primary Dependencies**: Firebase SDK v10 (modular), firebase-admin v12, firebase-functions v2, esbuild (bundler), idb v8
**Storage**: Firestore (document store) + Firebase Auth (identity + custom claims) + IndexedDB (client cache)
**Testing**: Vitest (unit), @firebase/rules-unit-testing (integration), Playwright (E2E)
**Target Platform**: Modern evergreen browsers (client) + Cloud Functions Node.js 20 (server)
**Project Type**: Web application (client-rendered SPA admin + serverless functions)
**Performance Goals**: Admin page load <2s (SC-001), Cloud Function cold start <500ms
**Constraints**: No custom server (Constitution I), Blaze plan required (R2), 1-10 CMS users
**Scale/Scope**: 63+ site pages, 4 role levels, 1-10 CMS users, 3 content collections + 4 new collections

## Key Design Decisions

| Decision | Rationale | Constitutional Anchor |
|----------|-----------|----------------------|
| esbuild, not Vite | Zero config, 1 command, already in deps | XIV (Simple First) |
| Dual storage (claims + Firestore) | Rules need fast claim check; UI needs queryable Firestore | VII, R3 |
| Audit log as version history | No extra collection; reconstruct from entries | XIV, VI |
| All-content editor access | Simpler than per-doc assignment at 1-10 users | XIV, INS-RBAC-002 |
| Bootstrap as security invariant | Cannot lock out all super_admins | VII, INS-RBAC-003 |
| Client-side idle timeout | No server-side revocation needed at this scale | XIV, R5 |
| Legacy claim fallback | Smooth migration from `admin: true` to `role: x` | VI, VIII |
